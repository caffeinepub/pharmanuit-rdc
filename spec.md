# PharmaNuit RDC

## Current State

Full-stack ICP application with:
- Public homepage listing validated pharmacies with search and proximity features
- Pharmacy detail page with call, directions, view number, view address buttons
- Pharmacist registration/login (Internet Identity based) with pending/active/suspended account statuses
- Pharmacist dashboard showing views count and pharmacy info editor
- Admin page (route `/admin`) with 3 tabs: Pharmacies management, Accounts management, Statistics
- Admin access via hidden route `/admin-pharmanuit-dashboard` → `/admin`
- Backend: Motoko with `Pharmacie` type (nombreVues), `Utilisateur` type, `StatutPharmacie` (validee/enAttente/suspendue), `StatutCompte` (actif/enAttente/suspendu)
- Call tracking: NOT yet implemented
- Statistics per pharmacy (calls today, calls this month): NOT yet implemented
- Pharmacist registration currently uses Internet Identity only; NO email/password/pharmacy-name/address/phone/hours fields during signup (only nom, email, password, role)
- `StatutPharmacie` has no "rejected" status yet
- Admin cannot "Reject" a pharmacy (only validate or suspend)
- Admin cannot permanently delete a user account (only suspend/activate)

## Requested Changes (Diff)

### Add

1. **Call tracking system** — new backend storage for call events per pharmacy with timestamps. New function `enregistrerAppel(pharmacyId)` callable by anyone (no auth required). New query functions `getNombreAppelsAujourdhui(pharmacyId)` and `getNombreAppelsMois(pharmacyId)` for pharmacist dashboard and admin.

2. **"Rejected" status for pharmacies** — add `#rejetee` variant to `StatutPharmacie`. Add admin action `rejeterPharmacie(pharmacyId)` to set status to `#rejetee`. Rejected pharmacies are NOT visible to public.

3. **Admin: reject pharmacy account** — new `rejeterCompte(userId)` function sets user `statutCompte` to a new `#rejete` value, or reuse `#suspendu` with a separate rejection flag. Simplest: add `#rejete` to `StatutCompte`.

4. **Pharmacist registration extended fields** — the inscription form must collect: pharmacy name, address, phone number, opening hours. These are stored on the `Utilisateur` record (new fields: `nomPharmacie`, `adressePharmacieInscription`, `telephoneInscription`, `horairesInscription`) so admin can see them when reviewing pending accounts. When admin approves, pharmacist dashboard auto-creates the pharmacy entry using these stored fields.

5. **Auto-create pharmacy on approval** — when admin calls `approuverCompte(userId)`, backend sets user status to `#actif` AND auto-creates a `Pharmacie` record (status `#enAttente` initially, then `#validee`) linked to that user if they don't already have one.

6. **Admin statistics dashboard** — extend existing StatistiquesTab to show:
   - Total pharmacies, approved, suspended, pending, rejected counts
   - Total calls generated by app (all time)
   - Per-pharmacy table: name, clicks (nombreVues), calls today, calls this month

7. **Pharmacist dashboard call stats** — add to pharmacist dashboard:
   - Calls today for their pharmacy
   - Calls this month for their pharmacy
   - These auto-refresh

8. **Call tracking trigger on frontend** — when user taps "Appeler" button on PharmacyDetailPage, call `enregistrerAppel(pharmacyId)` on backend before opening tel: link.

9. **Admin: permanently delete user account** — add `supprimerUtilisateur(userId)` function. Admin can fully remove a user account from the database. Also removes their associated pharmacies.
